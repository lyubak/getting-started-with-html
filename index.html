The Basics of the Web and HTML

The web is a bunch of computers that communicate with each other.

When a person goes to a web page like www.google.com, their computer sends a HTTP Request to a server. The server finds the appropriate HTML document and sends it back to the user's computer where a web browser interprets the page and displays it on the user's screen.

HTML – Hyper Text Markup Language

HTML is made up of:

  • Text content -> what you see
  • Markup -> what it looks like
  • References to -> e.g. images + videos
  • Links to other pages

HTML Markup

(<name>content</name>) - ELEMENT

<name> - opening tag

</name> - closing tag

Types of HTML Elements

  • Inline - an inline element occupies only the space bounded by the tags that define the inline. Generally, inline elements may contain only data and other inline elements. By default, inline elements do not begin with new line.
  • block - block elements form an "invisiible box" around the content outside of them. By default, block elements begin with a new line

Tree-like structure of HTML

The "tree-like structure" comes from the fact that HTML elements can have other elements inside of them.
In a browser, this structure shows up as a series of nested boxes. There are boxes inside of boxes inside of boxes, and so on...

Relationships between indented HTML and boxes

When you read an HTML document as text, you see a wave of changing indentations going up and down the left side of the document. The more indented an element is, the more deeply nested it's corresponding "box" is.

The Box Model

Box Sizing

  1. HTML elements are boxes and each box has 4 components
  2. Because the are so many components to each box, it can pften be hard to get the box just right
  3. There are two techniques you can use to help deal with sizing issues:
    1. Set sizes in terms of percents rather than pixels
    2. Set the box-sizing attribute to border-box for every element
  4. Different browsers work slightly differently. Sometimes this causes different browsers to display the same code differently

Box Positioning

  1. Divs are block elements (as opposed to inline), so by default they take up the entire width of a page
  2. Adding the rule display: flex; to the appropriate CSS will override this behavior and let divs appear next to each other

Cascading Style Sheets (CSS)

CSS is code written to control the "style" of HTML elements.

Descendant elements may inherit CSS property values from any ancestor element enclosing them.

Cascading

The rules are applied not only to the elements they directly match but also to all those elements’ child elements.

Avoid Repetition

  1. The most specific rule is applied to every element
  2. Inheritance is the mechanism by which properties are applied not only to a specified element, but also to its descendants.
  3. Relies on the document tree. Descendant elements may inherit CSS property values from any ancestor element enclosing them.
  4. Descendant elements inherit text-related properties, but box-related properties are not inherited.

Properties that can be inherited are color, font, letter-spacing, line-height, list-style, text-align, text-indent, text-transform, visibility, white-space and word-spacing.

Properties that cannot be inherited are background, border, display, float and clear, height and width, margin, min- and max-height and -width, outline, overflow, padding, position, text-decoration, vertical-align and z-index.

NOTE: If a chilld element has multiple overlapping rules defined for it, the more specific rule takes effect.

CSS Styling of a Page

There are several methods for applying styles to a HTML document.

Method 1

Write CSS in the <head> of the HTML. This method is suitable for small projects.

Method 2

Link the HTML to a separate CSS file. This method helps to stay more organized when working on big projects.

Method 3

Write styles inline with the HTML using the "style" attribute. This method is not recommended becasuse it causes a lot of code repetition.

CSS Selectors

  • ID Selectors - the most powerful type of selector in terms of CSS specificity.
  • Class Selectors.
    • well supported in all browsers
    • you can add multiple classes (just separated by a space) on HTML elements
    • there are JavaScript things you can do specifically for manipulating classes
  • Tag Selectors
    • most useful when changing properties that are unique to that HTML element
    • also in reset stylesheets where you are specifically trying to unset styles that browsers apply to certain elements

    Note: Don't rely on tag selectors too much though. It's typically more useful to have a class define styling that you can use on any HTML element.
